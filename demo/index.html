Clocklet

An opinionated clock-style timepicker.

Philosophy, Opinions, Goals

Users canâ€™t control the mouse very well.
Timepickers MUST NOT disable keyboard input.
Click targets used frequently SHOULD be large enough.
Less surprise; follows the mental model of users.
Timepickers that look like clocks can be intuitive.
- Most datepickers look like calendars.
Many clock-style timepickers have a state that represents either "entering hour" or "entering minute".
I never want to care the state.
Both hour and minute should be displayed and selectable at any time.
Less clicks, less key inputs
See also: Date-Input Form Fields: UX Design Guidelines

Features

Keyboard and numpad friendly (by Lenientime)
Autocomplete - e.g. "1"->"01:00", "12"->"12:00", "1234"->"12:34"
Up and down arrow key support
Mouse and touch friendly
Sexy and stylish- not confident
